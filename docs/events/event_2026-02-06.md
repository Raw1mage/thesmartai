#### 功能：修正 Model Activities 排序切換時的游標位置

**需求**

- 當 Model Activities 頁面在「常用模型優先」模式下，點選模型導致其位置跳到最頂行時，游標應跟隨跳動到該模型的新位置，而非停留在原處（導致沒選中的錯覺）。

**範圍**

- IN：`src/cli/cmd/tui/component/dialog-admin.tsx` 的模型選取與排序邏輯。
- OUT：不改變現有的排序規則或資料分組。

**方法**

- 在 `selectActivity` 執行後，觸發 `DialogSelect` 的游標重新定位（如果模型位置改變）。
- 或者，在 `local.model.set` 引起資料順序變動後，確保 `DialogSelect` 的選中狀態與資料同步。
