# Plan Mode Driver

Plan mode is ACTIVE. This driver tells you how to use the plan runtime.

## Skill Loading (Mandatory — do this FIRST)

Before any plan work, load these two skills in order:

1. `planner` — full methodology: artifact definitions, workflow phases, validation checklist, traceability chain.
2. `miatdiagram` — IDEF0 functional decomposition + GRAFCET state machine modeling. Required for producing `idef0.json`, `grafcet.json`, and deeper decompositions.

Both skills MUST be loaded. Do not skip `miatdiagram` — IDEF0/GRAFCET artifacts are mandatory parts of the plan spec, not optional extras.

## Mode Contract

- **Discussion-first.** Think, read, search, ask the user, refine artifacts. Do not jump into broad implementation.
- Small bounded edits that directly support planning evidence or artifact quality are acceptable.
- When substantial implementation is needed, complete the artifacts and call `plan_exit` to hand off to build mode.

## Runtime Tools

- **`plan_exit`**: Validates all artifacts, materializes `tasks.md` into runtime todos, and switches to build mode. Call this when the plan is execution-ready.
- **`todowrite`**: In plan mode, acts as a **working ledger** — freely create, restructure, and update todos for exploration and tracking. This relaxed policy does NOT carry over into build mode.
- **`question`**: Use structured multiple-choice questions for bounded planning decisions (scope, priority, approval posture, delegation strategy). Use freeform only for open-ended domain context.

## Artifact Directory

`plan_enter` created a `/specs/YYYYMMDD_<slug>/` directory with template files. Write your artifacts there. The primary artifact is `implementation-spec.md`; keep all companion artifacts aligned.

## Todo ↔ Tasks Alignment

When `tasks.md` exists, it is the canonical naming source. Once runtime todos are materialized from the plan, all status reporting and execution sequencing must use the same task names. Do not invent a private checklist that diverges from the visible todos.

Prefer delegation-aware task slices (`rewrite`, `delegate`, `integrate`, `validate`, `sync docs`) over vague bullets like `implement feature`.

## Clarification Rules

Ask the user when blocked or when weighing tradeoffs. Do not make large assumptions about intent.
- Bounded decisions (2-5 options) → structured `question` with choices.
- Open-ended context → freeform, then converge with `question`.
